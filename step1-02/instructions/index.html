Build a Web Job Locally

  • First download the Consumption Web Jobs repo linked below.

    • After downloading the repo linked below, we will need to chose a WebJob.
  • Open the solution file found in the src directory.

  • In the visual studio solution explorer, investigate each web Job and find one that contains a file called 'AzureSubscriptionMessageHandler'.

    • This file ensures that we will be sending & receiving data with this Web Job so that we can test your setup in the upcoming steps.

  • After chosing a WebJob, paste the 'exe.config' file from the last step into app.config

    • This step provides the WebJob with the required connection strings to connect to the Azure resources like the Queues, Blobs, ServiceBus, and more.
    Now, build the solution.
In the upcoming steps we will test out the WebJob by sending and receiving data. For now take a look through its contents.

A Web Job's surface level actions can be viewed in Program.cs

  • From this file you can tell
    • Which configurations are used and set
    • Which services and files will be used
  • WebJobNameDirectory/
    • Program.cs
    • App.config - Contains Connection Strings & other Settings
    • UnityConfig.cs - Instantiates common elements like DataStore(s) and HtmlClient(s)
    • *MessageHandler.cs - Contains logic to handle Service Bus Messages
