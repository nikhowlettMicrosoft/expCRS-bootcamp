Sending & Receiving Data With Service Bus & Service Bus Explorer

Service Bus is an integral type of Azure Resource that acts as Pub-Sub system.
We Will be using Service Bus Explorer to send and receive messages in this portion of the bootcamp.

  • Download the Service Bus Explorer linked below which will be used to send messages to use in our Web Job.
  • Then either create a new Service Bus or use the provided test environment Service Bus below. (Aptly named by its creators)

  • You will need to get the connection string to send messages. To get this, enter the Shared access policies under the Settings Submenu in the left hand columnar menu on the Service Bus Resource in the Azure Portal.
  • Copy the Primary Connection String for the RootManageSharedAccessKey Policy and use that string as the connection string in Azure Service Bus Explorer.

  • In AzureServiceBusExplorer, right click on the subscription of the topic that is mentioned in the WebJob either in Program.cs, App.config, or AzureSubscriptionMessageHandler.cs
  • Press Send message, and then start
  • Now that you've sent Data, your Visual Studio window should hit the breakpoint we created in step 3!
Congratulations we have sent data to our WebJob which was successfully received!
